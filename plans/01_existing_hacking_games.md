# 기존 해킹/코딩 게임 레퍼런스 정리

이 문서는 **장르 레퍼런스에서 검증된 재미 요소**를 추출하고, 우리 게임에 적용할 때의 주의점(특히 자동화 과몰입, 콘텐츠 소모 속도)을 정리한다.

---

## 1) Uplink 계열에서 배울 것

### 장르적 특징(요약)
- 플레이어는 “프리랜서 해커”로서 **계약(contracts)**을 수행하고 돈을 벌며, 점점 더 어려운 시스템을 뚫기 위해 **하드웨어/소프트웨어를 구매**한다.
- 분위기는 “엄청 리얼”이라기보다 **영화적/스타일리시한 해킹**을 지향한다.
- (우리 게임 적용) *리얼리즘보다 ‘전개/긴장/보상’이 우선*이라는 방향성에 근거 제공.

### 우리 게임에 가져오기 좋은 요소
- **계약 기반 경제 루프**: 미션 → 보상 → 장비/프로그램 업그레이드 → 더 어려운 미션.
- **장비 성장 체감**: CPU/RAM, 네트워크 장비, 계정/권한 도구 등.
- **“도구(프로그램)” 중심 진행**: 새로운 프로그램을 얻는 것 자체가 콘텐츠.

### 주의점(우리식 개선 포인트)
- 스킬/장비만으로 뚫리는 느낌이 강해지면 단조로워질 수 있음 →  
  “도구를 사는 것” + “도구를 직접 코딩으로 튜닝/조합”이 같이 작동해야 함.

---

## 2) Hacknet 계열에서 배울 것

### 장르적 특징(요약)
- 터미널 기반, 유닉스풍 파일시스템과 명령 사용.
- 일반 흐름: **스캔 → 방어 요소 파악 → 대응하는 프로그램 실행 → 루트 획득 → 파일 목표 달성**.
- “프로그램이 RAM을 먹는” 형태의 제한이 게임 플레이의 전략이 된다(어떤 툴을 동시에 띄울지 고민).

### 우리 게임에 가져오기 좋은 요소
- **도구를 ‘실행 중 메모리 점유’로 모델링**: RAM이 곧 빌드/세팅의 재미가 됨.
- **긴장 시스템(추적/카운트다운/탐지)**: 유저가 “빨리/은밀하게” 중 무엇을 선택할지 고민.
- **파일시스템 기반 목표**: 로그/백업/설정 파일에 단서가 있고, 침투 후 수집/조작이 핵심.

### 주의점(우리식 개선 포인트)
- “미리 정해진 절차대로만” 하면 퍼즐 풀이로 고정될 수 있음 →  
  유저 스크립트/자동화를 통해 **다양한 공략 루트**를 만들게 설계.

---

## 3) Bitburner 계열에서 배울 것

### 장르적 특징(요약)
- **프로그래밍 기반 자동화**가 핵심(스크립트로 거의 모든 플레이를 자동화 가능).
- 리소스(특히 RAM) 제약이 “실행 가능한 스크립트 규모”를 결정한다.
- 장점: 코딩하는 재미가 직접 성장/효율로 이어짐.
- 단점: 자동화가 극단으로 가면 게임이 “혼자 돌아가는 방치”가 되기 쉬움.

### 우리 게임에 가져오기 좋은 요소
- **툴을 ‘프로그램 파일’로 취급**: 얻고, 뜯고, 고치고, 배포하고, 최적화.
- **RAM/CPU 업그레이드가 곧 ‘실행 가능한 전략의 확장’**.
- “스캐너/브레이커/디코더/디플로이어” 같은 도구 생태계 구축.

### 자동화 과몰입을 막는 장치(권장)
- **탐지/로그**: 너무 빠르고 큰 자동화는 Trace를 크게 올려 리스크 증가.
- **이벤트/상호작용 미션**: 사람이 판단해야 하는 분기(사회공학, 퍼즐형 단서, 시간 제한 이벤트).
- **단계별 인증/권한**: 완전 자동화가 어렵도록(예: 일부 승인/토큰은 타이밍/상황 의존).

## 4) 레퍼런스에서 공통으로 보이는 “재미의 뼈대”

1) **정보 수집이 곧 퍼즐**  
   스캔/배너/로그/파일로 “무슨 방어가 있는지” 추론한다.

2) **리소스 제약이 전략을 만든다**  
   RAM/CPU/시간(추적 카운트다운/감시 강화)이 선택을 강제한다.

3) **도구(프로그램)가 곧 콘텐츠**  
   도구를 얻고/업그레이드하는 순간이 새로운 가능성(새 루트)이다.

4) **긴장감(탐지/추적)**  
   ‘안전하게 오래’ vs ‘빠르게 위험하게’의 트레이드오프가 몰입을 만든다.

---

## 5) 우리 게임의 차별화 포인트(이 문서의 결론)

- 기존 게임은 “도구를 얻어서 쓰기”가 중심인 반면,  
  우리는 **도구의 내부가 MiniScript로 공개**되어 있어 유저가 직접:
  - 도구 성능 개선(더 빠르게/더 은밀하게/더 적은 RAM)
  - 자동화(루틴화/이벤트 처리)
  - 새로운 공격 방법을 조합(도구 체이닝/조건 분기)
  를 할 수 있다.
